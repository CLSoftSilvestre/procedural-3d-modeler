# Procedural 3D Modeler for three.js

A browser-based, **node-graph procedural 3D modeling app** that exports
**production-ready, parameterized three.js code** (plus glTF/GLB and a re-editable
graph). Build parametric geometry visually; ship lightweight generators, not baked assets.

## Project documents (read in this order)

1. **[PROMPT.md](PROMPT.md)** — the vision, the problem, the end-game. The *why*.
2. **[ARCHITECTURE.md](ARCHITECTURE.md)** — stack, data model, engine & codegen. The *how*.
3. **[DEVELOPMENT_PLAN.md](DEVELOPMENT_PLAN.md)** — phases, milestones, tasks, session
   log. **The single source of truth.** The *what next*.

## Working convention

- `DEVELOPMENT_PLAN.md` is authoritative. Read it at the start of every session and
  **update it at the end of every session** (task checkboxes, Session Log, Next Up).
- Architectural decisions are recorded as ADRs in `ARCHITECTURE.md`.
